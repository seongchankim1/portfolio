동시성 문제

문제점: 다수의 사용자가 동시에 입찰하는 과정에서 데이터 충돌 및 값 갱신 오류 발생

대안 목록:

  • 데이터베이스 락: 트랜잭션 내 동시성 보장, 부하 증가
  • synchronized 키워드: 단일 JVM에만 유효
  • Redis Redisson: 분산 환경 FairLock 제어

선택: Redis Redisson FairLock 적용

구현:

public  T lock(String lockName, Supplier operation) {
  RLock lock = redissonClient.getFairLock(lockName);
  try {
    boolean isLocked = lock.tryLock(WAIT_TIME, LEASE_TIME, TIME_UNIT);
    if (!isLocked) throw new IllegalArgumentException("입찰 대기시간 초과");
    return operation.get();
  } finally { lock.unlock(); }
}

결과: 10명 이상의 동시 입찰에서도 데이터 무결성 100% 유지

분산락 설정 문제

문제점: Repository Delete 시 @Transactional 충돌로 롤백 및 락 해제 실패

대안 목록:

  • 기능별 락 관리: 코드 중복 증가
  • AOP Annotation 처리: 공통 관심사 분리

선택: @DistributedLock 애노테이션 도입

@Around("@annotation(distributedLock)")
public Object handleLock(ProceedingJoinPoint jp, DistributedLock dl) throws Throwable {
  String key = createDynamicKey(jp, dl.key());
  return lockManager.lock(dl.prefix()+":"+key, () -> jp.proceed());
}
